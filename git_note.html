3.Git环境变量

git config --list:显示所有Git环境变量

git config --list --[system|global|local]:显示不同层级Git环境变量。local：只针对当前repo。global：针对当前登陆用户的所有repo。system：针对该机器所有登陆用户的repo

比如git config --list --local user.name:显示当前repo拥有者的名字。注意，local只有在切换到git repo路径下才能使用。

git config --local user.name 'yourname':可以将用户名设置为yourname

4.创建Git Repo

如果没有建立项目，可以切换到想要建立项目的根目录git init repo-name 

cd root-dir
git init repo-name /会在root-dir下建立一个repo-name为名的文件夹

如果已建立项目，但是想要纳入Git管理，可以切换到项目目录git init 

cd repo-dir
git init

如果最终想要在github上管理，也可以现在github上创建项目，然后在本地git clone 

cd root-dir
git clone link-to-the-repo

5.Git的基本操作

Git包括工作区和暂存区(Stage)

将文件从工作区加至暂存区git add file-name-to-add

将文件从暂存区删除git rm --cached file-name-to-unstage

git status查看现在缓存区的状态

git add -u将所有曾经被add到stage的文件（这些文件已被track），如有更新，一同添加到stage

git reset --hard抹去所有工作区，暂存区的文件变更

git commit -am'commit message'将工作区的变更越过暂存区直接添加

git commit --amend更新commit message

git rm old-file-name new-file-name重命名文件

git log查看git日志

git log --oneline查看git日志，每个commit只有一行作为summary

git log -n4查看最近的4条git日志

git log --all显示所有branch的日志

git log branch-to-view显示所有branch-to-view的日志

git log --graph用图形化方式展示日志

git branch -v查看所有本地分支

git branch -d branch-to-delete删除所选本地分支，如果无法删除，可以是使用-D

git checkout -b new-branch-name创建并切换到新分支

git branch -av展示所有active的分支

git branch new-branch-name hashkey-of-commit为hashkey-of-commit创建一个分支，命名为new-branch-name

merge branch-A to branch-B 

git checkout branch-B /checkout到你想要并入的分支
git merger branch-A /合并branch-A到branch-B

git diff commit1 commit2比较commit1和commit2

git cat-file hashkey-for-git-object -t查看hashkey-for-git-object的类型，git object可能是commit，tag，tree，blob

git cat-file hashkey-for-git-object -p查看hashkey-for-git-object的内容

detached HEAD:分离头指针，是指当前工作（HEAD）没有挂钩任何分支，当切换分支时，git会提示detached HEAD，此时可以选择联系到分支或者放弃储存变更，如果在切换分支时，没有做相应处理，commit将不会纳入分支文件管理，之后可能会被清理
